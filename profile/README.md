## Our overall structure

Regardless of the project we're working on, every game will be split into 3 big parts:

### Visual Assets (artist)

This includes **everything** that the player will see on screen, 3d or 2d in-game models, such as (if applies)

- Characters
- Vehicles
- Environmental assets (trees, water, animals atc)
- Game store assets
- Icons
- UI elements

This section rest on the shoulders of [alebahia](https://github.com/alebahia).

### Audio Assets (artist)

Audio is such an important part of any game, this section includes

- Any in game sound
  - Steps
  - Engines
  - Crash sounds
  - Activation/deactivation sound effects
  - Ambient music
- Voice acting
- UI sounds (buttons and menu interactions)

### Programming (engineering)

Any part that requires code, scripting, any programming language or similar such as, but not limit to:

- Game mechanics
  - Driving
  - Flying
  - Character movement
  - logic
  - Systems
    - Inventory
    - Combat
    - Health / Damage
- Interactions
- Behaviour
- UI/UX programming
- Networking
- Websites

This part scumble on the back of [fish](https://github.com/th3fishMk)

## Our AI policy

Whether we like it or not, AI is everywhere nowadays specially with companies like Microsoft and Microsoft and, did I mentioned Microsoft already? pushing AI down our troats. Disgusting! Truth is, many of us are here for the love of the game, we don't care about getting 10x productivity nor about being left behind because some random dude in the internet is using x or y prompt to get the best slop possible.

> **Note** we use the generic term "AI" to refer at any "LLM" or "GenAI" models such as (but not limited to) `chatGPT`, `gemini`, `claude`, `nano banana` and similar models.

AI is a great tool though  
We are not strictly against AI, after all a tool is a tool, and we can take advantage of it. we are, however, opting for a more responsible use of AI

That being said, we agree at the following use cases of AI

### What are we allowed to do

Members of AFI Games are allowed to use AI tools for `research`, `clear doubts`, and `find resources`.

Examples of this are actually simple, we are replacing old good Google search for AI

Lets suppose we're working on a truck simulator game

We are allow to:

```markdown
Q: What are the speed limits for trucks on the united states? Are they different from cars and other vehicles?
A: Some very long text...
```

Simple enough, we are also allowed to

```Markdown
Q: How many speeds does a truck gearbox regularly have? Are electric truck actually becoming popular?
```

[Read this conversation](https://gemini.google.com/share/9c96af336dcf)

### What are are **not** allow to do

Use the direct output of an LLM, even if the topic is complicated, we agree to spend the necessary time to understand and craft what we're delivering to the player.

```markdown
Input: An icon for a game studio
Output: An image "ready" for production
```

[here is this example on gemini](https://gemini.google.com/share/fa9ac4e02fba)

```markdown
Q: Create a unity script for a vehicle controller, add realistic gearbox, suspension, and differential system
```

```markdown
Q: Create engine start and stop sounds for a truck
A: <audio-file>
```
